# Memory Before Action

## The Rule
Before your first substantive response about any known project, you MUST:

1. **Check if memory context was injected.** If it was, acknowledge what you already know before doing anything else.
2. **If no context was injected**, but the user mentions a project topic, search memory yourself before responding.
3. **Never read files, give opinions, or take action** until you have confirmed what you already know from prior sessions.

## Why This Exists
Sessions that skip memory recall treat prior work as if it never happened. Each time wastes the user's time and erodes trust.

## What "Acknowledge" Means
- Reference specific prior decisions, not vague summaries
- Show you understand where the project left off
- Only THEN proceed to the current request
