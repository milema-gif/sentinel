# Session End Save

## The Rule
Before ending ANY session (saying "done", wrapping up, or when context is getting low), you MUST:

1. **Save key decisions/discoveries** to persistent memory
2. **Save a session summary** with: Goal, Discoveries, Accomplished, Next Steps, Relevant Files

## What to Save (minimum per session)
- Any decision made (architecture, convention, tool choice)
- Any bug fixed (include root cause + fix)
- Any user preference or correction learned
- Any non-obvious discovery or gotcha found
- Files significantly changed

## Self-Check
Ask yourself: "If I start a fresh session tomorrow on this project, what would I need to know that isn't in the code or git history?"

## Why This Exists
Multiple sessions have saved 0 observations. Every session should leave breadcrumbs for the next one.
